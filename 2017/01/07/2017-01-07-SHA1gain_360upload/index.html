SHA获取及360加固上传

发表于 2017-01-07 | 更新于 2017-01-07 | 分类于 Android

关于SHA获取及app360加固上传的操作说明。

1. SHA1的获取

cmd-> cd .android ->调试版本使用 debug.keystore，命令为：keytool -list -v -keystore debug.keystore;
发布版本使用 apk 对应的 keystore，命令为：keytool -list -v -keystore apk 的 kenstore。链接1

若keytool 命令不存在，需要设置环境变量，在 .android 下，直接输入keytool直至显示命令为止；然后开始上面的做法。
解决方法二，链接2
把 C:\Users****.Android 目录下的 debug.keystore 拷贝到 K:\Program Files\jdk\bin下即可

当然这只是debug版本SHA1的获取，

问题是debug版和发布版的SHA1是不同的，网上说是相同，是错误的。
实际上，最简单正确的获取方式应该是在app中获取；

重新回去注册api_key,让我发现在输入SHA1码的输入框旁边有一个问号，点开刚还看到相应的问题，进去就发现了这个方法：
**public static String sHA1(Context context) {….}
这个方法返回的就是高德所需要的SHA1码(可以log显示出来)，debug版本和release版本都输出显示出来(两者SHA1不一致),把它填到高德申请控制台里，得到最终的key值即可解决问题。

2. 360加固上传

  • 一开始，我把AS调试产生的debug.apk直接丢到360加固助手中签名apk,发现一只发现如图所示的问题；(jarsigner无法对jar进行签名)
  • 几经周折，试了许久仍然不行，后来直接通过AndroidStudio自带工具签名apk即可，build->generate signed apk完成release.apk；
  • 然后，将得到的release.apk拖到360加固保中直接加固应用，发现最终成功加固，功能完美实现。
