开发问题罗列02

发表于 2017-08-15 | 更新于 2017-09-02 | 分类于 problems

罗列一些平时开发过程中遇到的问题及解决方法。

1.Git相关(170808)

  1. pull conflict错误, soa-lib中.gitgnore文件conflict

    解决：在相关文件夹里删掉.gitgnore文件后(不要在eclipse里删除)，重新再pull 恢复正常
    哪个文件有conflict就删除哪个文件，提前做好备份即可。

2.js相关

在js中，0==’’或者0==“”，这两种类型是一样的。要区分两者，需要加上typeof的字符串类型过滤。(0808)

(data[p]==""&&typeof(data[p])=="string")

3.smartAs项目相关

  1. UI层中返回String类型数据出错(0810)

    解决：

    Resource.get(“services/soa/contractapproval/index”,
    data=>{console.warn(data);}
    );//ResourceGetIndex

    因为 此回调函数中data是数组形式， 导致UI层必须return list或者数组。
  2. JSX组件导入(170814)

（A）

/*[[
   import "web/contract/ContractForm.jsx"
   import "web/component/StringUtil.js"
]]*/
上面的（A）写法并不是单纯的注释， 是真实具体的引入写法。
（B）const StringUtil =web.emp.component.StringUtil;引入对象Obj。 其中（A）（B）写法要共同使用才能正确引用别的文件内容。 （C）install("web.emp.contract.ContractDetail",function($S){... C项一定要跟（B）项一一对应上，才能正常响应。 ### 4. 写法问题 1. 特殊写法
{getFieldDecorator('contactname')()} 其中，disabled里是单的中括号，不是双的。
2. 错误写法
onlyReadflag=true disabled={this.props.onlyReadflag} 不能这样写，会报错。 修改：(值为1即true) disabled={this.props.onlyReadflag==1} disabled={"true"} ### 5. 报错 1. handleSave中props报错
handleSave在constructor中没有bind的问题
2. service.update不成功
data没有id值了, 主要是建立form时没有填充id值，在最后一行添加一行代码即可： {getFieldDecorator('id')()} 3. 有外键关联的时候删除出错
在外键的删除属性上设置CASCADE模式即可。 ### 6. 引入组件的问题 /*[[ import "web/StringUtil.js" ]]*/ const StringUtil =web.StringUtil; ### 7. web启动报错， 1. jdbc类型写错，INT 要改成 INTEGER ;
2. 报错一直显示没有的 控件或属性，即停留在上一次代码的状态
解决方法1： 可能是系统出错，把原有组件复制粘贴重命名，重新启动运行可以恢复正常
解决方法2： Google刷新按钮右键，清空内存刷新即可。
3. 报错，某某方法或属性undefined-->可能是相对应的 对象null导致的。 ### 8. MyBatis多参数查询 一种解答链接
更好的解决方法： 在FileDao中，多注解一一对应即可： 

List<FileInfo> getByBusinessFileId( @Param("businessId") long businessId, 
@Param("fileType") String fileType);
//FileDao.xml里：
WHERE T.BUSINESS_ID= #{businessId} and T.FILE_TYPE=#{fileType}

9. 网页中部分乱码问题

设置eclipse全空间编码一致，UTF-8

Windows-> Preferences-> General-> Workspace-> UTF-8
