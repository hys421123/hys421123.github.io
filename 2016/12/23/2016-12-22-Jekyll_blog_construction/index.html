2.搭建GitHub博客遇到的问题及解决

  1. 环境变量设置后，并不能成功在cmd根目录下显示基本功能，一直都是内部无此命令。解决： 可能系统没反应过来，通过重启恢复正常。

    could not find a valid gem jekyll,unable to download a data from https://ruby.gems.org
    解决： gem sources -a http://rubygems.org

    搭建本地 myBlog时， jekyll new myBlog出现问题。Error:bundler
    <font color="blue">解决：</font> gem install bundler <br>
    ***最重要的一个问题： could not find gem minima解决： 缺少什么，
    就去gem install什么，比如 gem install minima即可!!!

  1. github博客没有同步更新，–>肯定是本地md文件代码有错误,解决这个显示错误即可同步更新了。(重要注意点)
    但是仍然存在同步更新问题，换个浏览器新打开就好了，貌似是浏览器缓存的问题吧。

  2. github博客显示貌似不支持 ##(二级标题) –>是支持二级标题的，而且 h2,h3显示也没问题。 一级就不用了。

  3. 使用prose时，保存出现 error while loading data from github… –>. 这是github授权问题，应该重新退出登录prose账号授权一下，即可恢复正常。

  4. github博客界面没更新， 查看hys邮件，发现GitHub page build failure，以此解决问题即可。 最常见问题： date为空，没写具体时间！！

  5. index.html中的title是 网址tab里的title，不是网址页面里的 title。

  1. 本地网站错误，因为混入 中文，导致编码错误。
    修改之后一直没反应 —> bak文件删除即恢复正常！！

  2. {{ … }}表示， 包裹输出内容
    {{ %… %}} 是服务端程序代码

  3. jekyll 3.3 默认没有paginator, 要自己在 gemfile和config里面添加即可。
    config –> gems: [jekyll-paginate]

    paginate: 20 
paginate_path: "page:num"

  4. config 要调整时， 需要重新启动服务器。 jekyll server

  5. 图片显示不出来 –> 路径出现问题
    src=’assets/home/background.jpg’ –> src=’/assets/home/background.jpg’ 注意根目录 ‘/‘
