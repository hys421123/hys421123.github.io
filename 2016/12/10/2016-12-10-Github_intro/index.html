Github Introduction

发表于 2016-12-10 | 更新于 2016-12-10 | 分类于 github

初识GitHub，程序员世界的开始！

1.Git、GitHub简介

Git 是一款免费、开源的分布式版本控制系统，版本控制是一种记录一个或若干文件内容变化，以便将来查阅特定版本修订情况的系统，对于软件开发领域来说版本控制是最重要的一环，而 Git 毫无疑问是当下最流行、最好用的版本控制系统。 11

GitHub 主要提供基于 git 的版本托管服务。也就是说现在 GitHub 上托管的所有项目代码都是基于 Git 来进行版本控制的。

git push origin master 意思就是把本地代码推到远程 master 分支；

git pull origin maste 意思就是把远程最新的代码更新到本地。一般我们在 push 之前都会先 pull ，这样不容易冲突。

2.GitHub网站操作

1.repository 仓库 一个项目对应一个仓库。

issue 提的问题、bug等；

fork 分叉 本质上是在原有项目的基础上新建了一个分支，分支的概念后面会在讲解Git的时候说到），他就可以随心所欲的去改进，但是丝毫不会影响原有项目的代码与结构；

watch 观察， 那么以后只要这个项目有任何更新，你都会第一时间收到关于这个项目的通知提醒；

3.关于GitHub客户端操作

向 GitHub提交项目

链接

首先要通过设置ssh key连接GitHub和本地计算机

设置ssh: 设置详情

  1. 打开git bash;

    2.输入ssh，若有提示，说明本机已经安装ssh;

    3.输入 ssh-keygen -t rsa 生成两个文件id_rsa和id_rsa.pub;

    4.复制id_rsa.pub的内容到GitHub上的ssh key中；

    5.输入 ssh -T git@github.com 进行测试, 若出现successfully,则添加成功。


    出现问题的解决: 问题解决

    我的问题，运用shh -T -v git@github.com查看具体出错信息，发现缺少github_rsa.pub文件。再看相关文件夹，发现文件又补上了。
    再次运行ssh -T git@github.com即可成功。

提交更新，直接在本地仓库的项目中修改代码即可。

1.先要在github上创建个人仓库

2.然后在客户端上clone刚创建的仓库到本地

3.接着打开clone到本地的文件夹 将需要上传的项目添加进去这里以FirstRespository文件夹为例，里面放你想要上传的项目

4.添加之后回到github客户端,先要commit(只有在changes变化完之后才能sync)，然后点击右上角的sync按钮同步提交即可。 注意，有的commit，还要先去掉.git文件夹,是任何文件中都不能有这个!!才能commit。

4.AS提交代码

整个过程的链接

  1. 从as中下载github代码，
    先要设置ssh
    链接
  2. 检查AS setting 中的Git.exe是否运行正常。
  3. AndroidStudio中从云端导入项目 setting-> Version Control-> GitHub-> 登录账户即可
    然后，File->New-> Projects from Version Control-> GitHub->
    Clone Repository-> 选择你需要的GitHub仓库工程-> Test Successful之后，点击Clone即可。 可能 AS中会显示unregistered vcs root detected,add root即可。
  4. 把云端代码向as中更新， VCS-> Git-> Pull
  5. 将as更新的代码向 云端提交 Git->Push

5.其他注意的

提交更新时，说明用英文写出 所做的一些修改,因为时间github上默认有标出，因此不用以时间作为修改说明。
